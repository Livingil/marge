## Что заполнить перед подключением

1. Юрданные:
   - точное название юрлица/ИП
   - ИНН, ОГРН/ОГРНИП
   - юридический и фактический адреса
   - support email/phone
2. Политики:
   - публичная оферта (url/текст)
   - политика обработки персональных данных (url/текст)
   - политика возвратов (url/текст)
3. Фискализация:
   - схема онлайн-кассы/ОФД
   - ставки НДС для SKU
   - СНО
4. Backend:
   - включить валидацию webhook-signature
   - идемпотентные ключи на `create payment`
   - журнал статусов `pending/succeeded/canceled`
5. Frontend:
   - экран результата оплаты (`success/fail/pending`)
   - блокировка повторного списания при повторном нажатии
   - отображение чеков/квитанций в истории покупок

## Переменные окружения

Backend (`backend/.env`):

- `YOOKASSA_ENABLED`
- `YOOKASSA_SHOP_ID`
- `YOOKASSA_SECRET_KEY`
- `YOOKASSA_RETURN_URL`
- `YOOKASSA_WEBHOOK_SECRET`
- `YOOKASSA_VAT_CODE`
- `YOOKASSA_TAX_SYSTEM_CODE`

Frontend (`frontend/.env`):

- `VITE_YOOKASSA_ENABLED`
- `VITE_YOOKASSA_RETURN_URL`

## Порядок включения

1. Закончить рекламный запуск и замерить retention/ARPDAU.
2. Заполнить реквизиты и юридические документы.
3. Подключить тестовый магазин YooKassa.
4. Прогнать тест-кейсы оплаты/возврата/повторного webhook.
5. Включить `YOOKASSA_ENABLED=true` и `VITE_YOOKASSA_ENABLED=true` только после финальной проверки.
